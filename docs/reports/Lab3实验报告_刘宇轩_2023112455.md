# Lab3实验报告（CodePilot 版）

## 实验要求
围绕 CodePilot 完成代码审查、黑盒测试、白盒测试与覆盖率验证。

## 1. 代码审查
- `validate_task_request`：补充输入归一、空值校验与 `mode` 白名单。
- `evaluate_operation_risk`：补充中文风险词与默认确认策略。
- `run_task_session`：确保 plan / auto 主流程、失败回退与重试上下文可追踪。

## 2. 黑盒测试
### 2.1 定义
黑盒测试从用户视角验证系统输出，不关注内部实现。

### 2.2 测试内容
| 测试项 | 重点 | 预期结果 | 对应文件 |
|---|---|---|---|
| 输入合法性 | 任务描述、工作目录、模式是否符合要求 | 非法输入被拒绝，合法输入通过 | `tests/unit/test_models.py` |
| 模式分流 | `plan` / `auto` 行为是否正确 | `plan` 停在确认，`auto` 进入执行 | `tests/unit/test_runtime_sprint2.py` |
| 风险识别 | 危险文本是否触发高风险提示 | 高风险命令被拦截或要求确认 | `tests/unit/test_runtime_sprint2.py` |
| 真实联调 | 仓库上下文、planner、命令执行是否闭环 | 能输出可追踪结果与失败信息 | `tests/unit/test_runtime_sprint2.py` |

### 2.3 黑盒回答模板
> 黑盒测试主要站在用户视角，验证系统面对不同输入时是否返回符合预期的输出。比如合法任务能否创建成功，空描述是否被拒绝，非法模式是否报错，plan 模式是否停在待确认状态，auto 模式是否进入执行状态。

## 3. 白盒测试
### 3.1 定义
白盒测试关注内部逻辑、分支和异常路径。

### 3.2 测试内容
| 目标函数 / 流程 | 覆盖路径 | 预期结果 | 对应文件 |
|---|---|---|---|
| `validate_task_request()` | 空描述、超长描述、空目录、超长目录、非法模式、合法返回 | 各分支都被验证 | `tests/whitebox/test_whitebox_branches.py` |
| `evaluate_operation_risk()` | 空字符串、高风险、中风险、低风险 | 风险等级判定正确 | `tests/whitebox/test_whitebox_branches.py` |
| `PlanExecutionController.start_task()` | `plan` / `auto` 分支切换 | 分别进入等待确认或准备执行 | `tests/unit/test_runtime_sprint2.py` |
| `run_task_session()` | planner 成功、超时回退、unsafe 命令过滤、语法损坏回退、失败重试 | 回退与重试链路可追踪 | `tests/unit/test_runtime_sprint2.py` |

### 3.3 白盒回答模板
> 白盒测试主要看代码内部结构，针对条件分支、异常路径和关键逻辑做覆盖。比如输入校验函数有哪些分支、风险识别函数如何区分高/中/低风险、计划控制器在 plan 和 auto 两种模式下分别走哪条路径。

## 4. CodePilot 主流程（文字版）
1. 用户输入任务。
2. `validate_task_request` 做输入归一与合法性校验。
3. `run_task_session` 创建会话并汇聚上下文。
4. `GitHubRepoClient` 读取仓库摘要与相关信息。
5. `DeepSeekPlannerClient` 生成计划；若超时或异常，则回退到本地启发式规划。
6. `evaluate_operation_risk` 识别风险等级并决定是否需要确认。
7. `plan` 模式停在等待确认；`auto` 模式进入执行。
8. 执行文件读取、写入和编辑。
9. 运行测试、lint 或 build。
10. 输出 diff、日志与结果；失败时回传上下文并重试。
11. 保存 history / snapshot。

## 5. 老师常见追问与简答
| 问题 | 简答 |
|---|---|
| 黑盒测试到底测什么？ | 测用户可见行为，看输入输出是否符合预期。 |
| 白盒测试到底测什么？ | 测内部分支和异常路径，看关键逻辑是否覆盖。 |
| 为什么要同时做黑白盒？ | 一个保证可用，一个保证可控。 |
| CodePilot 重点测什么？ | 输入校验、plan/auto 分流、风险识别、执行回退、真实联调。 |
| 测试结果说明什么？ | 核心闭环可用，关键逻辑可解释，失败时有防线。 |
| 测试有没有局限？ | 主要覆盖核心流程，复杂并发和大规模样本还可继续扩展。 |

## 6. 结论
本实验围绕 CodePilot 完成了代码审查、黑盒测试、白盒测试与覆盖率验证。黑盒测试覆盖用户输入、模式分流和真实联调；白盒测试覆盖关键函数分支、异常路径与回退机制。整体上，系统的核心路径已经具备较好的可测试性与可解释性。
