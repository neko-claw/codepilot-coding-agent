# 项目开发文档

## 1. 项目名称
CodePilot：从零实现的智能 Coding Agent 开发平台

## 2. 项目背景
随着大模型与软件工程工具的发展，编程辅助系统正在从“代码补全”走向“任务级代理执行”。传统编程助手通常只能在局部文件上给出建议，而真正的 Coding Agent 需要具备以下闭环能力：
- 理解自然语言开发需求
- 扫描仓库结构并建立上下文
- 输出可确认的执行计划
- 修改一个或多个文件
- 运行命令并验证结果
- 输出变更、日志、风险提示与回退能力

课程实验的目标不是做一个大而全的商业产品，而是做一个可落地、可演示、可验证的原型系统。因此，本项目采用“先核心闭环、后增强能力”的开发策略。

## 3. 项目目标
1. 支持自然语言输入开发任务
2. 扫描仓库结构并识别关键配置文件
3. 在执行前生成步骤化计划
4. 支持多文件代码修改
5. 支持命令执行与结果回传
6. 支持 Diff、日志、风险确认与回退
7. 形成可测试、可度量、可迭代的系统骨架

## 4. 目标用户
- 软件工程课程项目开发者
- 学生程序员与个人开发者
- 需要在小型仓库中执行自动化修改的用户

## 5. 系统范围
### 本阶段纳入范围
- 任务输入
- 仓库扫描
- 任务规划
- 文件编辑
- 命令执行
- 结果验证
- 风险控制
- 历史记录与回退

### 本阶段暂不重点实现
- 多人协同编辑
- 企业级权限体系
- 云端分布式调度
- 完整 IDE 插件生态

## 6. 产品功能分解
| 模块 | 功能说明 | 输出 |
|---|---|---|
| 任务交互模块 | 接收需求、展示会话状态 | 任务请求对象 |
| 仓库理解模块 | 扫描目录树、关键文件与入口 | 仓库摘要 |
| 任务规划模块 | 将自然语言需求转为执行步骤 | 结构化计划 |
| 代码编辑模块 | 创建、修改、删除文件 | 文件变更集 |
| 命令执行模块 | 执行 build/test/lint 命令 | 终端日志 |
| 验证与回退模块 | 确认结果、失败处理与回滚 | 验证报告 |
| 历史记录模块 | 保存会话和任务轨迹 | 历史任务列表 |

## 7. 用户故事主线
核心主线为：
`US-01 任务输入 → US-02 仓库理解 → US-03 计划确认 → US-04 代码修改 → US-05 命令执行 → US-06 Diff 展示`

辅助能力包括：
- `US-07` 危险操作确认
- `US-08` 执行日志记录
- `US-09` 失败修复与重试
- `US-10` 历史会话与任务记录
- `US-11` 执行边界控制
- `US-12` 一键回退

## 8. 目录规划
```text
src/codepilot/
├── core/        # 通用数据模型、请求与状态对象
├── planner/     # 任务拆解与计划生成
├── executor/    # 命令执行、日志采集、结果汇总
├── safety/      # 风险识别、确认与边界控制
├── storage/     # 任务记录、快照与回退
└── ui/          # 原型界面层或接口层
```

4. 测试设计与测试用例文档

6. 基础测试与覆盖率配置

## 10. 风险与应对
| 风险 | 影响 | 应对策略 |
|---|---|---|
| 大模型结果不稳定 | 计划与修改质量波动 | 引入确认机制、日志和回退 |
| 命令执行存在破坏性 | 误删文件、污染环境 | 执行白名单与风险确认 |
| 多文件修改难以验证 | 缺陷定位困难 | 强制输出 Diff 和测试结果 |
| 时间有限 | 功能做不全 | 先实现高优先级闭环，再做增强 |

## 11. 开发原则
- 先实现最小可运行闭环，再扩展能力
- 所有关键操作必须可验证、可回退、可解释
- 测试设计与文档交付同步推进，不后补
- 原型实现优先服务于课程验收，但保留后续演化空间
