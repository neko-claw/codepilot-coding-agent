# 测试方案、测试用例与覆盖度设计

## 1. 测试目标
本项目测试的目标不是单纯证明“代码能跑”，而是验证 Coding Agent 在关键业务路径上的正确性、稳定性、安全性与可解释性。测试设计覆盖以下层次：
1. 静态检查：Ruff、Pylint
2. 单元测试：核心函数与对象校验
3. 黑盒测试：围绕需求输入、任务规划、命令执行等用户视角验证
4. 白盒测试：围绕条件分支、异常路径、风险分类逻辑验证
5. 覆盖度测试：统计语句覆盖、分支覆盖与核心路径覆盖

## 2. Pylint / Ruff 测试思路
### 2.1 Ruff
Ruff 用于快速发现：
- 未使用导入
- 未定义变量
- 风格不一致
- 明显可自动修复的问题

建议命令：
```bash
ruff check .
```

定位：作为**快速反馈层**，每次提交前都执行。

### 2.2 Pylint
Pylint 用于发现：
- 命名不规范
- 设计复杂度过高
- 参数过多
- 可读性与潜在坏味道

建议命令：
```bash
pylint src/codepilot
```

定位：作为**质量审查层**，每轮迭代合并前执行。

### 2.3 组合原则
- Ruff 负责快
- Pylint 负责细
- 两者互补，不互相替代

## 3. 黑盒测试设计

## 4. 白盒测试设计
白盒测试关注代码内部逻辑、判断分支和异常路径。

### 4.1 `validate_task_request()` 白盒分析
主要分支：
1. 描述为空
2. 描述超长
3. 目录为空
4. 目录超长
5. 模式非法
6. 全部合法并返回对象

因此至少需要 6 类测试用例，覆盖所有判断出口。

### 4.2 `evaluate_operation_risk()` 白盒分析
主要分支：
1. 输入为空字符串
2. 命中高风险关键词
3. 命中中风险关键词
4. 未命中任何风险关键词

至少需要 4 类测试，覆盖全部路径。

### 4.3 白盒测试用例
| 用例ID | 目标函数 | 覆盖路径 | 预期 |
|---|---|---|---|
| WB-01 | validate_task_request | 描述为空分支 | 抛异常 |
| WB-02 | validate_task_request | 描述超长分支 | 抛异常 |
| WB-03 | validate_task_request | 目录为空分支 | 抛异常 |
| WB-04 | validate_task_request | 目录超长分支 | 抛异常 |
| WB-05 | validate_task_request | 模式非法分支 | 抛异常 |
| WB-06 | validate_task_request | 合法返回分支 | 返回 TaskRequest |
| WB-07 | evaluate_operation_risk | 空字符串分支 | medium + confirm |
| WB-08 | evaluate_operation_risk | 高风险分支 | high + confirm |
| WB-09 | evaluate_operation_risk | 中风险分支 | medium + confirm |
| WB-10 | evaluate_operation_risk | 低风险分支 | low + no confirm |

## 5. 覆盖度设计
### 5.1 覆盖度指标
- **语句覆盖（Statement Coverage）**：每条可执行语句至少被执行一次
- **分支覆盖（Branch Coverage）**：每个判断的真/假路径都被执行
- **核心路径覆盖（Critical Path Coverage）**：对任务输入、风险判断、执行确认等关键路径进行覆盖

### 5.2 目标阈值
| 指标 | 目标值 |
|---|---|
| 语句覆盖率 | ≥ 85% |
| 分支覆盖率 | ≥ 80% |
| 核心模块覆盖率 | ≥ 90% |

### 5.3 覆盖度命令
```bash
pytest --cov=src/codepilot --cov-report=term-missing --cov-report=html
```

### 5.4 为什么覆盖率不能代替测试质量
覆盖率高只能说明“代码被执行过”，不能说明断言充分、场景真实。因此必须将：
- 黑盒测试（用户视角）
- 白盒测试（逻辑视角）
- 静态分析（代码质量）
结合起来，才能形成可信的质量评估。

## 6. 测试执行顺序建议
1. 先执行 Ruff，快速发现低级问题
2. 再执行 Pylint，发现设计与规范问题
3. 再执行 Pytest 单元测试
4. 最后执行带 Coverage 的完整测试

推荐命令：
```bash
ruff check .
pylint src/codepilot
pytest --cov=src/codepilot --cov-report=term-missing
```

## 7. 测试结论模板（可直接放入实验报告）
本项目测试方案从静态分析、黑盒测试、白盒测试和覆盖度统计四个层面展开。静态分析使用 Ruff 与 Pylint 识别代码规范与质量问题；黑盒测试基于等价类划分和边界值分析验证任务输入与风险提示等外部行为；白盒测试围绕关键函数的条件分支与异常路径构造测试用例；覆盖度测试用于量化核心模块的语句覆盖与分支覆盖情况。该方案能够较完整地验证 Coding Agent 原型的正确性、鲁棒性和可维护性。
